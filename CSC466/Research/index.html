Stage Two - Edge Detection

The next step of the process is to detect the brush strokes from an image. The process that I used for this includes multiple image manipulation processes, but the main act of the show is edge detection.

  1. The image is converted to greyscale, because it make the operations faster and easier to manage.
  2. The image is blurred to make features stand out more.
  3. The image is then dilated to accentuate the brush strokes within.
  4. The image then undergoes the edge detection process, this happens via another discrete convolution, with a different matrix.
  5. A k-means of ```K=2``` is then applied to the image so that edges will link together better.
  6. The image is then transformed from a greyscale image to a binary 2 bit image via a threshold function.
  7. The edges are then thinned and any loose pixels are cleaned up.

Stage Three - Feature Extraction

The library that I use to perform the more complicated aspects of this project contains a k-means function, but it dosen't perform how I want it to. For this, I modified it so that it would output an image based on what the K values was. For feature extraction of the brush strokes, k-means does all of the heavy lifting out of getting nice clusters of pixels.
The edge detection is overlayed on top of a slightly processed reference image. While this does not extract all the strokes that I want it to, and there are large clusters which impact the outcome of the data, this is good for now.
